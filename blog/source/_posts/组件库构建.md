## monorepo

monorepo追求的是在一个仓库中管理多个模块，每个模块有独立的package.json管理各自依赖，同时在项目根目录下可以通过命令安装或升级模块依赖，并提供了一个模块共享的node_modules。

### yarn workspace

yarn workspace 是实现monorepo的一种方式。

```ts
{
  "private": true,
  "workspaces": ["packages/*"]
}

```
